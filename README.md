## 装 Windows 之前，先搞清楚配置要求

这步很多人跳过，然后装完发现卡得没法用。

内存至少要 2GB。Windows 10 本体在什么都不跑的情况下大约占 1.5GB 内存，你还需要留点给系统进程用。1GB 的套餐装上去能跑，但鼠标移动都会卡，CPU 使用率长期高企，还可能触发搬瓦工的 CPU 限制机制——这个情况下 VPS 会被自动降速。

硬盘方面，Windows 安装完大约占 15-20GB，搬瓦工 KVM 基础套餐给的是 20GB SSD，真的只够装系统，几乎没有剩余空间可用。如果你打算在 Windows 上跑点东西，建议考虑配置高一点的套餐。

总结：**装 Windows 的最低配置是 2GB 内存 + 40GB 以上硬盘**，低于这个就别为难自己了。

## 方式一：通过 KiwiVM Mount ISO 安装 Windows

这是目前最干净的方式，不需要在命令行里运行任何脚本，全程图形化操作。

搬瓦工在 KiwiVM 控制面板的 Mount ISO 功能里已经内置了 Windows 镜像，包括：

- `win11_iot_ltsc_2024`（Windows 11 物联网企业长期服务版，2024）

这些镜像带 VirtIO 驱动，在 KVM 虚拟化环境下性能表现比普通版本好一些。

注意：**搬瓦工只提供安装介质，Windows 的正版授权需要你自己解决。** 这点要心里有数。

### 具体安装步骤

1. **登录 KiwiVM 控制面板**。进入搬瓦工账户后台，找到你的 VPS，点击右侧的"KiwiVM Control Panel"按钮。

2. **停止 VPS**。在 KiwiVM 首页点击"Stop"或"Force Stop"，等到状态变成"Stopped"。

3. **找到 Mount ISO 功能**。KiwiVM 首页往下滑，可以看到 Mount ISO 的下拉列表，从里面选择你想要的 Windows 版本，比如`win10_ltsc_2021`。

4. **点击 Mount 挂载**。等面板显示"Great Success! ISO has been mounted"之后，挂载就完成了。这个过程需要几分钟，耐心等，期间页面会显示"Task in progress"，不要反复刷新或重复操作。

5. **启动 VPS 并打开 VNC**。点击"Start"按钮，VPS 会从刚才挂载的 ISO 启动，同时 KiwiVM 会自动弹出 VNC 控制台窗口（如果被浏览器拦截，手动允许弹窗）。

6. **在 VNC 里完成 Windows 安装**。VNC 窗口里就是 Windows 安装向导，按照提示一步步做：选语言、磁盘分区、等待安装完成。这个过程大概需要 20-40 分钟，视 VPS 性能而定。

7. **安装完成后 Unmount ISO**。非常重要！回到 KiwiVM 首页，点击"Unmount"取消挂载。不做这步的话，下次重启还会从 ISO 启动，进入安装界面。

8. **重启 VPS**。先 Force Stop，再 Start，VPS 就会从硬盘启动进入刚装好的 Windows 系统。

9. **用 RDP 连接**。Windows 安装完成后，开启远程桌面（RDP），用 Microsoft Remote Desktop 或者 mstsc 连接 VPS 的 IP 地址，端口默认 3389。

安装过程中的常见问题：VNC 响应慢是正常的，VNC 本身延迟就比较高，不是 VPS 的问题。如果 VNC 长时间黑屏，Force Stop 之后重新 Start 再试一次。

## 方式二：DD 包方式（Linux 改装 Windows）

这个方式历史更长，很多老教程都在用。原理是先装 Debian 或其他 Linux，然后在 Linux 里跑一个 DD 脚本，直接把整个磁盘用 Windows 镜像覆盖写入，重启之后就变成 Windows 了。

DD 这个词来自 Linux 命令`dd`（disk dump），表示磁盘级别的镜像写入。整个过程就是在 SSH 里执行一行命令，然后等待。

**这种方式的优点**：不依赖 VNC，用命令行就能完成；镜像可以选 Windows 7、Windows Server 等老版本。

**这种方式的缺点**：镜像质量参差不齐，来源复杂；脚本需要从外部拉取，如果镜像地址失效或下载慢，安装会中途失败；对新手不太友好，出问题不容易排查。

大致流程是：

1. 在 KiwiVM 里把系统装成 Debian（一键安装，几分钟完成）
2. SSH 登录 VPS，运行 DD 脚本
3. 脚本执行完后 SSH 会断开，属于正常现象
4. 等待约 15-60 分钟
5. 用 RDP 连接 VPS，默认用户名和密码视所用镜像而定

整个 DD 过程一般 15-30 分钟，如果超过一个小时还没动静，大概率是镜像下载卡住了。

说实话，如果你现在买的是搬瓦工在售套餐，优先用 **Mount ISO 方式**。

## 装完 Windows 之后，几个需要配置的地方

**开启远程桌面**：Windows 默认远程桌面（RDP）可能是关闭的，需要在系统属性里手动打开。路径：我的电脑 → 属性 → 远程设置 → 允许远程连接。

**防火墙放行 3389 端口**：Windows 防火墙默认会拦截 RDP 端口，需要手动添加入站规则，放行 TCP 3389。

**修改 RDP 默认端口**（可选但建议）：3389 是公认的 RDP 端口，暴露在公网上会被反复扫描和爆破。建议改成一个非标准端口，比如 33890，同时在防火墙里相应修改规则。

**关闭 Windows 自动更新**（可选）：VPS 在自动更新期间 CPU 和磁盘会跑满，如果你的 VPS 配置刚好够用，更新过程中可能出现 CPU 被限速的情况。建议手动控制更新时机。

**安装 VirtIO 驱动**：用 Mount ISO 方式安装的 Windows 镜像已经内置 VirtIO 驱动，这块不需要额外处理。如果你用 DD 方式，可能需要手动安装驱动，网卡才能正常工作。

## 常见问题解答

**Q：搬瓦工装 Windows 合规吗？官方支持吗？**

搬瓦工 KiwiVM 控制面板里已经内置了 Windows ISO 镜像选项，说明官方层面是支持这个操作的。但有两点要注意：搬瓦工只提供安装介质，不提供 Windows 授权；使用 VPS 的行为必须合法合规。

**Q：安装完 Windows 之后怎么连接？SSH 还能用吗？**

Windows 不支持 SSH（除非单独装 OpenSSH Server），连接方式换成 RDP（远程桌面协议）。客户端用 Windows 自带的远程桌面（mstsc）或 Mac 上的 Microsoft Remote Desktop，输入 VPS 的 IP 地址和端口 3389，加上设置的 Windows 账号密码即可。

**Q：VNC 安装过程中黑屏或卡住了怎么办？**

先等 5 分钟，很多时候是正常的初始化加载。等了还是黑屏的话，回到 KiwiVM 首页 Force Stop，然后重新 Start，再看 VNC 里是什么情况。如果多次都是黑屏，可能是 VPS 内存不够，Windows 启动时崩溃了，考虑换内存更大的套餐。

**Q：Unmount ISO 这步能不能省略？**

不能。不 Unmount 的话，VPS 每次重启都会从 ISO 启动，进入安装界面而不是你已经装好的 Windows。这一步必须做。

**Q：搬瓦工支持退款吗，如果买了觉得不合适？**

新用户购买后 30 天内可以申请全额退款，在 KiwiVM 后台直接操作，自动处理，不需要联系客服。这个政策降低了试错成本，想试试 Windows VPS 体验的可以放心买来用。

**Q：装了 Windows 之后还能换回 Linux 吗？**

可以，随时都能换。在 KiwiVM 里用 Install new OS 功能一键重装 Linux，选好发行版等几分钟就好，跟装 Windows 之前的操作完全一样。数据会清空，所以换之前记得备份。
